Kaprodi

Ringkasan Peran

Kaprodi adalah operator akademik utama dalam TA Cloud. Peran ini mengelola data master, memvalidasi proposal, memantau progres skripsi semua mahasiswa, menetapkan pembimbing/penguji, menyetujui permohonan sidang, meninjau nilai akhir, dan menangani permintaan buka kunci nilai.

Fokus utama Kaprodi: validasi awal, penugasan reviewer, kontrol fase, dan monitoring akhir.

Dashboard

Dashboard Kaprodi menampilkan kartu statistik dan grafik fase berdasarkan periode terpilih.

  • Proposal Diajukan: jumlah proposal yang menunggu persetujuan.
  • Menunggu Assign: proposal yang sudah siap reviewer tetapi pembimbing belum ditetapkan.
  • Permohonan Sidang: permintaan sidang dari dosen yang masih menunggu keputusan Kaprodi.
  • Review Dokumen Final: skripsi yang sudah masuk tahap pemeriksaan dokumen akhir.

Menu Pengajuan Proposal

Menu ini dipakai untuk memvalidasi proposal awal mahasiswa.

Langkah 1
Buka daftar pengajuan

Masuk ke menu Pengajuan Proposal. Gunakan filter reviewer status atau periode bila diperlukan.

Langkah 2
Tinjau proposal

Klik proposal mahasiswa. Pastikan dokumen proposal sudah ada dan metadata skripsi sesuai.

Langkah 3
Approve atau Reject

Jika proposal layak lanjut, pilih Approve. Jika ada masalah administratif atau isi belum sesuai, pilih Reject.

Proposal yang di-approve biasanya menjadi kandidat untuk tahap penugasan pembimbing dan/atau sidang proposal sesuai alur sistem.

Menu Skripsi

Menu ini adalah pusat kendali skripsi per mahasiswa.

Sub-menu dan aksi utama

  • List Skripsi: cari, filter, dan buka detail.
  • Detail Skripsi: melihat ringkasan mahasiswa, fase, reviewer, dan dokumen.
  • Proposal: meninjau dokumen proposal dan aksi approve/reject proposal.
  • Bimbingan: melihat semua catatan bimbingan.
  • Detail Bimbingan: membaca satu catatan/sesi bimbingan secara spesifik.
  • Reviewer Search: mencari dosen untuk assignment.
  • Assign Pembimbing: menetapkan pembimbing 1/2.
  • Assign Penguji: menetapkan penguji 1/2.
  • Hapus Reviewer: membatalkan assignment jika salah pasang.
  • Logbook: unduh catatan bimbingan sebagai dokumen rekap.
  • Document Download: unduh dokumen revisi/final dari mahasiswa.
  • Update Status: memindahkan status skripsi sesuai progres.
  • Sidang Schedule: memperbarui jadwal sidang jika fitur dipakai.
  • Final Review Approve: menyetujui review dokumen final pada tahap akhir.
Urutan kerja Kaprodi paling umum: cek proposal → assign pembimbing → pantau bimbingan → proses permohonan sidang → pantau review final → cek nilai akhir.

Menu Permohonan Sidang

Permohonan sidang diajukan dari sisi dosen. Kaprodi berwenang menyetujui atau menolak.

  • Periksa siapa dosen pengaju.
  • Pastikan mahasiswa sudah berada di fase yang benar.
  • Pastikan reviewer sudah lengkap bila aturan prodi mensyaratkan.
  • Lakukan Approve atau Reject.

Menu Review Dokumen Final

Menu ini menampilkan skripsi yang sudah masuk tahap pemeriksaan dokumen akhir. Kaprodi dapat meninjau apakah dokumen sudah siap dianggap selesai.

Backend otomatisasi untuk publish ke library masih parsial. Gunakan menu ini sebagai kontrol administrasi internal.

Menu Nilai & Unlock Nilai

Ada dua area yang perlu dipahami:

  • Nilai: rekap akhir seluruh nilai yang sudah final.
  • Format Penilaian → Grades: halaman detail nilai berdasarkan template/skripsi yang juga memuat aksi Unlock.

Alur buka kunci nilai

  1. Dosen mengajukan permintaan buka kunci dari halaman penilaian.
  2. Kaprodi menerima notifikasi / melihat status permintaan.
  3. Kaprodi membuka halaman grade terkait.
  4. Kaprodi menjalankan aksi Unlock.
  5. Dosen dapat memperbaiki nilai dan menyimpan ulang.
Screenshot 4: Rekap nilai dan halaman unlock grade

Menu Dokumen Final

Menu Dokumen Final dipakai untuk mengelola template atau definisi dokumen final yang berlaku untuk periode tertentu.

  • Create: membuat template dokumen final baru.
  • Show: melihat detail template.
  • Edit: memperbarui informasi template.
  • Duplicate: menyalin template untuk periode baru.
  • Add Periode: mengaitkan template ke periode tertentu.
  • Remove Periode: melepas kaitan template dari periode tertentu.

Menu Master Data

Dosen

  • Index: lihat daftar dosen, cari, filter, dan buka detail.
  • Store/Update: tambah atau ubah data dosen.
  • Archive: arsipkan dosen tanpa menghapus permanen.
  • Restore: pulihkan dosen yang sudah diarsipkan.
  • Destroy: hapus permanen bila benar-benar tidak diperlukan.

Mahasiswa

  • Index: daftar mahasiswa aktif / arsip.
  • Store/Update: tambah atau ubah data mahasiswa.
  • Update Skripsi Status: ubah status skripsi dari level mahasiswa.
  • Archive/Restore/Destroy: kelola data historis mahasiswa.

Tahun Akademik

Dipakai untuk struktur kalender akademik. Tersedia aksi create, update, archive, dan destroy.

Periode

Dipakai untuk pengelompokan aktivitas skripsi per semester/periode. Tersedia aksi show, update, archive, dan destroy.

Format Penilaian

Dipakai untuk menyusun rubrik nilai. Tersedia aksi create, edit, duplicate, add/remove periode, lihat nilai per skripsi, dan unlock grade.

Menu Import Data

Gunakan impor CSV jika ingin memasukkan data dosen atau mahasiswa secara massal.

  1. Buka menu Import Dosen atau Import Mahasiswa.
  2. Unduh atau ikuti struktur template kolom yang diminta.
  3. Unggah file CSV.
  4. Tinjau ringkasan hasil impor: data berhasil, diperbarui, atau dilewati.

Halaman Monitoring Tambahan

  • Fase: board distribusi mahasiswa per fase. Berguna untuk monitoring cepat.
  • Keputusan: halaman ringkasan keputusan akhir sidang per skripsi.
  • Library: halaman monitoring item yang siap masuk arsip publik/library.
Ketiga halaman ini sudah tersedia, tetapi sebagian logika backend masih belum sepenuhnya otomatis.

Unduh / Arsip Internal

Pada sisi Kaprodi, fitur yang benar-benar tersedia saat ini adalah unduh logbook dan unduh dokumen dari halaman detail skripsi. Fitur ini berguna untuk arsip internal, rapat, audit akademik, atau pemeriksaan manual di luar sistem.

  • Logbook: mengunduh rangkuman catatan bimbingan mahasiswa.
  • Document Download: mengunduh proposal, revisi, atau dokumen final yang diunggah mahasiswa.
Catatan: dokumentasi ini tidak lagi menyebut fitur cetak khusus Kaprodi karena yang tersedia saat ini adalah unduh dokumen/logbook, bukan menu print tersendiri.
